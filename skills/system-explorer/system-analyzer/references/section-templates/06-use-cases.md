## Content Guidance

- "When to Use It" should describe workload characteristics, not just use case names
- "When NOT to Use It" is as important as when to use it — name concrete alternatives
- "Real-World Examples" should cite actual companies/projects with sources when possible
- This section bridges beginner and intermediate — start accessible, add technical detail
- Include inline links to case study blog posts and architecture write-ups
