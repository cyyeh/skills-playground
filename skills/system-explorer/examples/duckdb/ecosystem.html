Extensions

Core Extensions

DuckDB ships with a modular extension system maintained by the core team. Extensions add new file format support, remote access protocols, database connectors, and domain-specific functionality. Most core extensions auto-install the first time you reference them in a query, so you rarely need to manage them manually.

💡
Tip: You usually do not need to run INSTALL or LOAD commands. Just use the extension directly -- for example, SELECT * FROM 's3://bucket/data.parquet' auto-loads the httpfs extension. DuckDB handles the rest.

Bindings

Language Bindings

DuckDB provides official APIs across many languages, all built on the C/C++ core. The Python binding is by far the most popular, with deep integration into the pandas and Polars ecosystems. The WebAssembly binding brings DuckDB to the browser, enabling client-side SQL analytics.

🔭
Architecture insight: DuckDB's deep integration with pandas and Polars uses Apache Arrow as the common in-memory format. When you query a pandas DataFrame with duckdb.sql("SELECT ... FROM df"), no data is copied -- DuckDB reads the Arrow-compatible buffer directly. This zero-copy bridge is what makes DuckDB feel instant even on large DataFrames.
Cloud

MotherDuck (Cloud DuckDB)

MotherDuck is a cloud service built on DuckDB that extends the local-first experience to the cloud. The key idea is hybrid execution: your queries run locally or in the cloud depending on where the data lives, and you connect with a single ATTACH 'md:' command.

SQL 
-- Connect your local DuckDB session to MotherDuck
ATTACH 'md:';

-- Query a shared cloud database
SELECT * FROM md.shared_db.sales
WHERE region = 'APAC';

-- Hybrid: join local data with cloud data
SELECT l.name, c.revenue
FROM local_db.leads l
JOIN md.cloud_db.customers c
  ON l.email = c.email;
🔗
Hybrid Execution

Queries run locally or cloud-side depending on data location. DuckDB decides automatically.

👥

Share databases with your team for collaboration without managing infrastructure.

Offload heavy queries to cloud resources while keeping the local DuckDB experience.

ℹ️
Note: MotherDuck is a separate commercial service, not part of DuckDB itself. DuckDB remains fully functional without it. MotherDuck adds cloud storage, team collaboration, and the ability to scale compute beyond your local machine when needed.
