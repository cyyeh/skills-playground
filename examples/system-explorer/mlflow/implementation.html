Implementation Details

Storage backends, REST API internals, plugin architecture, and autologging mechanics.

Storage Backends

Backend Store (Metadata)

MLflow uses SQLAlchemy as the ORM layer, so any SQLAlchemy-compatible database works as a backend store.

BackendURI FormatNotes
SQLitesqlite:///mlflow.dbSingle-user, file-based. Good for development.
PostgreSQLpostgresql://user:pass@host/dbRecommended for production.
MySQLmysql://user:pass@host/dbVia PyMySQL or mysqlclient driver.
SQL Servermssql://user:pass@host/dbVia pyodbc driver.
File Store./mlrunsLegacy default. No registry support.
📊
Schema detail: The database includes tables for experiments, runs, params, metrics, tags, latest_metrics, registered_models, and model_versions. The latest_metrics table caches the final value of each metric for fast sorting and querying.

Artifact Store (Files)

Artifact storage is pluggable and completely separate from the metadata store. Each run gets a unique artifact directory: <artifact-root>/<experiment-id>/<run-id>/artifacts/.

bash 
# Production setup: PostgreSQL + S3
mlflow server \
  --backend-store-uri postgresql://user:pass@db-host:5432/mlflow \
  --default-artifact-root s3://my-bucket/mlflow-artifacts \
  --serve-artifacts \
  --host 0.0.0.0 \
  --port 5000

REST API Design

The MLflow REST API follows a flat, resource-oriented design under /api/2.0/mlflow/:

Key API Endpoints ▼
MethodEndpointPurpose
POST/experiments/createCreate a new experiment
GET/experiments/getGet experiment metadata
POST/runs/createStart a new run
POST/runs/log-parameterLog a single parameter
POST/runs/log-metricLog a single metric
POST/runs/log-batchLog params, metrics, tags in batch
GET/runs/searchSearch runs with filter expressions
POST/model-versions/createRegister a model version

The search API supports a SQL-like filter syntax: metrics.accuracy > 0.9 AND params.model_type = 'random_forest'

Plugin System

MLflow provides several extension points for custom integrations:

Tracking Store Plugins ▼

Implement a custom backend store by extending AbstractStore. Use case: proprietary databases or cloud-specific metadata stores.

Artifact Repository Plugins ▼

Extend ArtifactRepository to support custom file storage (e.g., a corporate artifact manager). Register via Python entry points:

python
# In pyproject.toml or setup.py
entry_points={
    "mlflow.artifact_repository": [
        "mystore=my_package.store:MyArtifactRepository"
    ]
}
Deployment Plugins ▼

Define custom deployment targets by implementing BaseDeploymentClient. This enables mlflow deployments create -t <target> for any infrastructure.

Autologging Internals

Autologging uses monkey-patching to intercept framework method calls. Here is the sequence:

⚙
Step 1: mlflow.autolog() detects which ML libraries are importable.
Step 2: For each supported library, patches the .fit() method with a wrapper.
Step 3: Before .fit() executes, the wrapper starts an MLflow run and logs constructor parameters.
Step 4: After .fit() completes, the wrapper logs metrics, the fitted model, and framework-specific artifacts.
Step 5: The wrapper restores the original method signature. If logging fails, training continues uninterrupted.

The implementation lives in mlflow/utils/autologging_utils/ with a safe_patch decorator that wraps all patched methods in try/except blocks to ensure training is never disrupted by logging failures.

Model Serialization Format

An MLflow model is stored as a directory containing:

text
model/
  MLmodel          # YAML: flavors, signature, metadata
  model.pkl        # Serialized model (or framework format)
  conda.yaml       # Conda environment specification
  requirements.txt # pip dependencies
  python_env.yaml  # Python version + env spec
🔒
Security: Since MLflow 3.10, pickle-free serialization is supported using torch.export (PyTorch) and skops (scikit-learn). Set MLFLOW_ALLOW_PICKLE_DESERIALIZATION=False to prevent loading pickle-based models, mitigating arbitrary code execution risks.

Model Signatures

Models can include a signature that defines the expected input/output schema, enforced at serving time:

python
from mlflow.models import infer_signature

signature = infer_signature(X_train, model.predict(X_train))
mlflow.sklearn.log_model(model, "model", signature=signature)
# Malformed input is rejected before reaching the model
