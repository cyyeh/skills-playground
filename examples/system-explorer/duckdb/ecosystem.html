Ecosystem & Integrations

Core Extensions

DuckDB's extension system provides modular functionality. Most core extensions auto-load when you first use them -- no manual installation required.

☁️
httpfs

Read/write to S3, HTTP, HTTPS, and Google Cloud Storage

📄
parquet

Parquet file reader/writer (bundled, auto-loaded)

📑
json

JSON file reader (bundled, auto-loaded)

🧊
iceberg

Apache Iceberg table format (v2 and v3 support)

🌍
spatial

GIS/geometry functions. GEOMETRY type is core in v1.5.0

🐘
postgres_scanner

Directly query PostgreSQL tables without importing data

💾
sqlite_scanner

Directly query SQLite databases -- great for migration

📊
excel

Read/write Excel (.xlsx) files with SQL

🏹
lance

Lance lakehouse format (new in v1.5.1)

MotherDuck (Cloud DuckDB)

MotherDuck is a cloud service that extends DuckDB beyond a single machine:

🔗
Hybrid Execution

Queries run locally or in the cloud depending on where the data lives. Use ATTACH 'md:' to connect.

👥
Team Collaboration

Shared databases let teams query the same data without each person maintaining a local copy.

💻
Web SQL Editor

Browser-based SQL editor for teams that want DuckDB's power without local installation.

🔭
Architecture insight: MotherDuck's hybrid execution is unique -- it keeps the local DuckDB developer experience while adding cloud compute for heavy queries. A query on a 10 MB local table stays local; a query on a 500 GB cloud dataset is offloaded to MotherDuck's servers.
