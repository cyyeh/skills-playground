Zero-Copy DataFrame Integration

Python
import duckdb
import pandas as pd

df = pd.DataFrame({
    'name': ['Alice', 'Bob', 'Charlie'],
    'department': ['Eng', 'Sales', 'Eng'],
    'salary': [120000, 95000, 110000]
})

# Query the DataFrame directly with SQL -- zero-copy!
result = duckdb.sql("""
    SELECT department, AVG(salary) as avg_salary
    FROM df
    GROUP BY department
""").fetchdf()

1. SQL Parser Entry Point

The parser wraps PostgreSQL's parser and transforms parse trees into DuckDB's internal representation.

void Parser::ParseQuery(const string &query) {
    Transformer transformer(options);
    PostgresParser parser;
    parser.Parse(query);
    if (parser.success) {
        transformer.TransformParseTree(
            parser.parse_tree, statements);
    }
}

2. Database Instance Initialization

The initialization sequence reveals the dependency order: file system first, then buffer manager, then catalog, and finally task scheduler.

C++ src/main/database.cpp View on GitHub ↗ 
void DatabaseInstance::Initialize(...) {
    Configure(*config_ptr, database_path);
    db_file_system = make_uniq<DatabaseFileSystem>(*this);
    db_manager = make_uniq<DatabaseManager>(*this);
    buffer_manager = make_uniq<StandardBufferManager>(
        *this, config.options.temporary_directory);
    db_manager->InitializeSystemCatalog();
}

3. Vectorized Operations Interface

C++ src/include/duckdb/common/vector_operations/vector_operations.hpp View on GitHub ↗ 
struct VectorOperations {
    static void IsNotNull(Vector &arg, Vector &result, idx_t count);
    static void And(Vector &left, Vector &right, Vector &result, idx_t count);
    static idx_t GreaterThan(Vector &left, Vector &right,
        optional_ptr<const SelectionVector> sel,
        idx_t count,
        optional_ptr<SelectionVector> true_sel,
        optional_ptr<SelectionVector> false_sel);
    static void Hash(Vector &input, Vector &hashes, idx_t count);
};
🔭
Design insight: Every operation accepts an idx_t count parameter, enabling batch processing. The selection-vector variants allow filter operations to produce selection vectors rather than copying data -- a key optimization that avoids unnecessary data movement.

5. Physical Plan Generator

C++ src/execution/physical_plan_generator.cpp View on GitHub ↗ 
PhysicalOperator &PhysicalPlanGenerator::ResolveAndPlan(...) {
    // Step 1: Resolve output types
    op->ResolveOperatorTypes();
    // Step 2: Convert bindings to physical indices
    ColumnBindingResolver resolver;
    resolver.VisitOperator(*op);
    // Step 3: Create the physical plan
    physical_plan = PlanInternal(*op);
    return physical_plan->Root();
}
