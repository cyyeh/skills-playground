Implementation Details

From installation to production deployment — hands-on code for every stage of working with DuckDB.

01 — Setup

Getting Started

DuckDB is available on every major platform and language. Pick your preferred installation method and you can be running analytical queries in under a minute. No server process, no configuration files, no Docker containers — just a library or binary.

Installation

bash 
# Python (most popular)
pip install duckdb

# CLI (Python-based installer)
pip install duckdb-cli

# macOS (Homebrew)
brew install duckdb

# Node.js
npm install duckdb

# Or download a binary from https://duckdb.org/docs/installation/

First Query (CLI)

sql 
$ duckdb
v1.5.1
Enter ".help" for usage hints.
Connected to a transient in-memory database.

D SELECT 42 AS answer;
┌────────┐
│ answer │
│ int32  │
├────────┤
│     42 │
└────────┘

First Query (Python)

python 
import duckdb

# In-memory (default)
con = duckdb.connect()

# Query a Parquet file directly
result = con.sql("SELECT region, SUM(amount) FROM 'sales.parquet' GROUP BY region")
print(result.fetchdf())  # Returns a pandas DataFrame

# Or use the relation API
con.sql("SELECT * FROM 'data.csv' WHERE age > 30").show()

Persistent Database

python 
# Creates/opens a file-based database
con = duckdb.connect('my_analytics.duckdb')
con.sql("CREATE TABLE events AS SELECT * FROM 'events_*.parquet'")
con.sql("SELECT event_type, COUNT(*) FROM events GROUP BY 1").show()
con.close()
02 — Configuration

Configuration & Tuning

DuckDB works well out of the box with sensible defaults. However, for large workloads or constrained environments, these settings let you control memory, parallelism, and I/O behavior.

sql 
-- Memory limit (default: 80% of system RAM)
SET memory_limit = '8GB';

-- Thread count (default: all cores)
SET threads = 4;

-- Temp directory for spill-to-disk (default: .tmp in current directory)
SET temp_directory = '/fast-ssd/duckdb-tmp';

-- Enable progress bar for long queries
SET enable_progress_bar = true;

-- Preserve insertion order (disable for faster bulk loads)
SET preserve_insertion_order = false;
💡
Performance tip: Setting preserve_insertion_order = false allows DuckDB to reorder rows during bulk inserts and COPY operations. This unlocks significantly faster parallel loading because worker threads no longer need to coordinate row ordering. Use this whenever insertion order does not matter for your workload.
03 — Python

Python Integration Patterns

DuckDB's Python API goes well beyond simple query execution. It offers zero-copy DataFrame interop, a chainable relation API, and direct export to multiple formats — all designed to fit naturally into data science and engineering workflows.

Zero-Copy Query on Pandas DataFrame

python 
import duckdb
import pandas as pd

# Load data into a pandas DataFrame
df = pd.read_csv('large_file.csv')

# Query the DataFrame directly with SQL -- no copy, no import
result = duckdb.sql("SELECT category, AVG(price) FROM df GROUP BY category")
print(result.fetchdf())

Relation API Chaining

python 
# Fluent API: read, filter, aggregate, sort, limit -- all chained
(duckdb
    .read_parquet('s3://bucket/data/*.parquet')
    .filter("year = 2025")
    .aggregate("region, SUM(revenue) AS total")
    .order("total DESC")
    .limit(10)
    .show())

Export to Different Formats

python 
# Read CSV, filter, write as Parquet -- one-liner ETL
duckdb.sql("SELECT * FROM 'input.csv' WHERE status = 'active'") \
      .write_parquet('output.parquet')

Persistent Analytics Store

python 
con = duckdb.connect('analytics.duckdb')
con.sql("""
    CREATE OR REPLACE TABLE daily_metrics AS
    SELECT date_trunc('day', timestamp) AS day,
           COUNT(*) AS events,
           COUNT(DISTINCT user_id) AS users
    FROM read_parquet('events_*.parquet')
    GROUP BY 1
""")
04 — SQL

Advanced SQL Features

DuckDB implements a rich SQL dialect that goes beyond standard SQL. It includes window functions, DuckDB-specific extensions like COLUMNS and EXCLUDE, time-series primitives, and syntactic sugar that makes analytical queries more expressive.

Window Functions

sql 
SELECT employee, department, salary,
       RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
FROM employees;

COLUMNS Expression (DuckDB-Specific)

sql 
-- Apply operations to multiple columns matching a pattern
SELECT MIN(COLUMNS('amount_*')), MAX(COLUMNS('amount_*'))
FROM transactions;

EXCLUDE / REPLACE in SELECT

sql 
-- Select all columns except the ones you don't need
SELECT * EXCLUDE (internal_id, debug_flag)
FROM users;

Recursive CTEs

sql 
WITH RECURSIVE org_tree AS (
    SELECT id, name, manager_id, 0 AS depth
    FROM employees WHERE manager_id IS NULL
    UNION ALL
    SELECT e.id, e.name, e.manager_id, t.depth + 1
    FROM employees e JOIN org_tree t ON e.manager_id = t.id
)
SELECT * FROM org_tree;

ASOF Joins (Time-Series)

sql 
-- Match each trade to the most recent price quote
SELECT t.*, p.price
FROM trades t ASOF JOIN prices p
ON t.ticker = p.ticker AND t.timestamp >= p.timestamp;

List Comprehensions and Lambdas

sql 
SELECT list_transform([1, 2, 3, 4], lambda x: x * x);
-- Returns [1, 4, 9, 16]
⚠️
Deprecation notice: The arrow syntax for lambdas (x -> x * x) is deprecated as of DuckDB v1.5.0. Use the lambda x: x * x syntax instead. The arrow syntax will be removed in a future release.

PIVOT / UNPIVOT

sql 
-- Rotate rows into columns
PIVOT sales ON region USING SUM(amount);

FROM-First Syntax (DuckDB-Specific)

sql 
-- Start with FROM instead of SELECT -- more natural for exploration
FROM sales SELECT region, SUM(amount) WHERE year = 2025 GROUP BY region;
05 — Extensions

Extension Management

DuckDB's extension system adds file format support, remote access, and domain-specific functionality. Most core extensions auto-load when needed, but you can also manage them manually.

sql 
-- Install and load an extension explicitly
INSTALL httpfs;
LOAD httpfs;

-- Most core extensions auto-load when needed:
SELECT * FROM 's3://bucket/data.parquet';  -- auto-loads httpfs

-- Configure S3 credentials
SET s3_region = 'us-east-1';
SET s3_access_key_id = 'AKIA...';
SET s3_secret_access_key = '...';

-- Or use the AWS credential chain
SET s3_use_credential_provider = 'credential_chain';

-- List installed extensions
SELECT * FROM duckdb_extensions();

06 — Deployment

Deployment Patterns

DuckDB adapts to a wide range of deployment scenarios — from quick throwaway scripts to production-grade ETL pipelines and embedded analytics services.

01

Local Analytics Script

The simplest pattern: a self-contained Python script that queries data on disk or in the cloud. No server, no setup — just run the script and get results.

python 
# One-off analysis -- no server, no setup
import duckdb

duckdb.sql("""
    SELECT country, COUNT(*) as users
    FROM 's3://analytics/users_*.parquet'
    GROUP BY country
    ORDER BY users DESC
""").show()
02

ETL Pipeline Stage

Read from multiple sources, join and transform with SQL, then write the result to an optimized format. DuckDB replaces heavyweight Spark jobs for datasets that fit a single machine.

python 
# Read from multiple sources, transform, write output
con = duckdb.connect()
con.sql("""
    COPY (
        SELECT a.user_id, a.event, b.plan_type
        FROM read_parquet('events/*.parquet') a
        JOIN read_csv('users.csv') b ON a.user_id = b.id
        WHERE a.timestamp > '2025-01-01'
    ) TO 'output/enriched_events.parquet' (FORMAT PARQUET, COMPRESSION ZSTD)
""")
03

Embedded in Web Application

DuckDB can power read-heavy API endpoints directly. Open the database in read-only mode and serve analytical queries with sub-second latency. The single-writer limitation is irrelevant here because the application only reads.

python 
# FastAPI endpoint backed by DuckDB
from fastapi import FastAPI
import duckdb

app = FastAPI()
db = duckdb.connect('analytics.duckdb', read_only=True)

@app.get("/metrics/{metric}")
def get_metric(metric: str, days: int = 30):
    result = db.execute("""
        SELECT date, value FROM metrics
        WHERE metric_name = ? AND date > current_date - ?::INTEGER
        ORDER BY date
    """, [metric, days]).fetchdf()
    return result.to_dict(orient='records')
04

Data Lakehouse with Iceberg

Query Iceberg tables stored on S3 with full support for time travel and schema evolution. DuckDB acts as a lightweight query engine against your lakehouse storage.

sql 
INSTALL iceberg;
LOAD iceberg;

-- Query an Iceberg table on S3
SELECT * FROM iceberg_scan('s3://lakehouse/db/sales');

-- Time travel: query a previous snapshot
SELECT * FROM iceberg_scan('s3://lakehouse/db/sales', version := '2');
🔭
Architecture insight: DuckDB's MVCC implementation with single-writer / multiple-reader semantics makes Pattern 3 (read-only web serving) particularly well-suited. Multiple HTTP request threads can query the same database file concurrently without any locking overhead, while a separate batch process handles writes during off-peak hours.
