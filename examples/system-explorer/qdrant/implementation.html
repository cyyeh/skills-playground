Implementation Details

Quick Start with Docker

bash 
# Run Qdrant with persistent storage
docker run -p 6333:6333 -p 6334:6334 \
  -v "$(pwd)/qdrant_storage:/qdrant/storage:z" \
  qdrant/qdrant:v1.17.1

This exposes the REST API on port 6333 and gRPC on port 6334, with data persisted to a local directory.

Python Client Usage

python 
from qdrant_client import QdrantClient
from qdrant_client.models import (
    Distance, VectorParams, PointStruct,
    Filter, FieldCondition, MatchValue,
    ScalarQuantization, ScalarQuantizationConfig, ScalarType,
    HnswConfigDiff,
)

# Connect (use prefer_grpc=True for production)
client = QdrantClient(url="http://localhost:6333")

# Create collection with quantization
client.create_collection(
    collection_name="products",
    vectors_config=VectorParams(
        size=768,
        distance=Distance.COSINE,
        on_disk=True,  # mmap storage
    ),
    hnsw_config=HnswConfigDiff(m=16, ef_construct=100),
    quantization_config=ScalarQuantization(
        scalar=ScalarQuantizationConfig(
            type=ScalarType.INT8,
            always_ram=True,
        ),
    ),
)

# Create payload index for filtering
client.create_payload_index(
    collection_name="products",
    field_name="category",
    field_schema="keyword",
)

# Upsert points in batch
client.upsert(
    collection_name="products",
    points=[
        PointStruct(id=1, vector=[0.1, 0.2, ...],
                    payload={"name": "Widget", "category": "electronics", "price": 29.99}),
        PointStruct(id=2, vector=[0.3, 0.4, ...],
                    payload={"name": "Gadget", "category": "electronics", "price": 49.99}),
    ],
)

# Search with filter
results = client.query_points(
    collection_name="products",
    query=[0.15, 0.25, ...],
    query_filter=Filter(
        must=[FieldCondition(key="category", match=MatchValue(value="electronics"))]
    ),
    limit=5,
    with_payload=True,
)

Configuration

Qdrant is configured via config.yaml or environment variables (pattern: QDRANT__SECTION__KEY).

yaml 
storage:
  storage_path: ./storage
  performance:
    max_search_threads: 0      # 0 = auto (CPU count)
    max_optimization_threads: 1

service:
  host: 0.0.0.0
  http_port: 6333
  grpc_port: 6334

cluster:
  enabled: false
  p2p:
    port: 6335

optimizers:
  memmap_threshold_kb: 200000   # Switch to mmap above 200MB
  indexing_threshold_kb: 20000  # Build HNSW above 20MB
  flush_interval_sec: 5

Production Deployment

Kubernetes (Helm)

bash 
helm repo add qdrant https://qdrant.github.io/qdrant-helm
helm install qdrant qdrant/qdrant \
  --set replicaCount=3 \
  --set config.cluster.enabled=true \
  --set persistence.size=100Gi

Performance Tuning Checklist

💡

Memory estimation: num_vectors x dimensions x 4 bytes x 1.5 (the 1.5x covers HNSW graph + metadata + temp segments)

Use mmap for large collections: set on_disk=True for vector storage

Enable scalar quantization with always_ram=True: quantized vectors in RAM, originals on disk

Start with 12 shards for future scaling flexibility (divides into 1, 2, 3, 4, 6, 12 nodes)

Batch ingestion: 1,000-10,000 points per request; disable indexing during bulk loads

Only index payload fields used in filters — each index consumes memory

Monitoring

Qdrant exposes Prometheus metrics at /metrics and provides a built-in web dashboard. Key metrics to watch:

MetricWhat to Watch
qdrant_points_totalTotal indexed points — verify ingestion completeness
qdrant_search_latency_secondsSearch latency histograms — alert on p99 spikes
qdrant_grpc_responses_totalRequest counts by status — watch for error rate increases
Optimizer statusPending optimizations indicate indexing backlog after bulk uploads
WAL segment countGrowing WAL indicates flush worker falling behind
