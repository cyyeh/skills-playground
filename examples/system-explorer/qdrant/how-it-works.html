How It Works

HNSW Graph Construction & Search

The Hierarchical Navigable Small World algorithm is the heart of Qdrant's search performance. When an immutable segment is built, the HNSW constructor assigns each vector a random level using an exponential distribution. Most vectors land at level 0 (densest layer); fewer reach higher levels.

Construction

Construction proceeds in two phases: a single-threaded bootstrap inserts the first batch of points to establish basic graph connectivity, then a parallel phase uses Rayon thread pools to insert remaining points concurrently. For each new point, the algorithm starts at the entry point, descends through upper layers, and at the point's assigned level, selects up to m neighbors using a diversity heuristic that rejects redundant short-range links.

Search

A search query follows a similar top-down traversal: enter at the highest layer, greedily walk toward the query vector, then at the base layer expand the search with beam width ef. The result is approximate nearest neighbors in O(log n) time.

💡
Tuning tip: Start with defaults (m=16, ef_construct=100). If recall is too low, increase ef_construct first (try 200), then m (try 32). At search time, increase ef (the search parameter) for higher recall at the cost of latency. Rule of thumb: ef should be at least 2x your top_k.

Filterable HNSW

Standard HNSW assumes all points are candidates. When payload filters restrict the search space, Qdrant's filterable HNSW integrates filtering directly into graph traversal: each candidate node is checked against the filter via payload indexes, but filtered-out nodes' graph links are still traversed as "stepping stones" to reach qualifying nodes.

The query planner estimates filter cardinality and selects the optimal strategy: high cardinality uses HNSW with in-traversal filtering; medium cardinality uses HNSW with expanded ef; low cardinality (below full_scan_threshold) skips HNSW entirely and brute-force scores the payload-matching points.

Quantization Engine

Qdrant implements three quantization strategies, each trading accuracy for memory and speed. All use a two-phase approach: quantized vectors produce a fast approximate ranking, then oversampled top candidates are re-scored against full-precision vectors.

Scalar Quantization4x compression, ~2x speedup
Product Quantization4-64x compression, slower per-query
Binary Quantization32x compression, 40x speedup
⚠️
Binary quantization caveat: Works best with high-dimensional embeddings (1024+ dimensions) from models like OpenAI's text-embedding-3-large. For lower-dimensional embeddings (384-768), binary quantization loses too much information — use scalar quantization instead.
