Ecosystem & Integrations

Framework Integrations

Qdrant has first-class integrations with the major AI/ML frameworks, allowing you to plug vector search into existing pipelines with minimal code changes.

langchain-qdrant

LangChain

The most widely used LLM framework. Supports dense/sparse/hybrid retrieval, metadata filtering, async operations, and multi-tenancy via collection partitioning.

llama-index

LlamaIndex

Simplifies data ingestion for LLM applications. Supports all Qdrant vector types, hybrid search with fusion strategies, and batch ingestion optimizations.

haystack

Haystack

Deepset's framework for building search and RAG pipelines. QdrantDocumentStore supports embedding retrieval, filtering, and batch operations.

semantic-kernel

Microsoft Semantic Kernel

Qdrant as a memory store for AI agents, enabling persistent semantic memories across conversation sessions in .NET and Python applications.

spring-ai

Spring AI

Vector store integration for Java/Spring Boot applications, enabling enterprise Java applications to leverage Qdrant vector search natively.

v1.17

Qdrant Edge

In-process Qdrant sharing the same internals and storage format. Enables edge computing, mobile apps, and testing without a separate server process.

DevOps & Observability

Prometheus + Grafana

Metrics exposed at /metrics endpoint. Monitor search latency, point counts, optimizer status, and gRPC error rates.

Helm Charts

Official Kubernetes deployment charts with configurable replicas, resources, persistence, and cluster mode.

Web Dashboard

Built-in UI for browsing collections, inspecting points, running test queries, and monitoring cluster health.

Snapshot API

Create and restore collection snapshots for backup, migration, and disaster recovery. Supports streaming uploads for bulk ingestion.
