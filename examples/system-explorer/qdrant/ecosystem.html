Local mode: The Python client supports a local mode that runs an embedded Qdrant instance in-process (backed by the same Rust engine via PyO3 bindings) — no server needed for development and testing.

Framework Integrations

LangChain

Available as the langchain-qdrant partner package. Provides a QdrantVectorStore class supporting dense, sparse, and hybrid retrieval modes. Handles document ingestion, similarity search, and MMR search natively.

LlamaIndex

Integrates as a vector store backend via the QdrantVectorStore class. Plugs into LlamaIndex's indexing and retrieval pipeline, enabling RAG applications with Qdrant as the knowledge store.

OpenAI

Qdrant serves as a memory backend for ChatGPT via the OpenAI retrieval plugin. Works seamlessly with OpenAI embedding models (text-embedding-3-small, text-embedding-3-large). Featured in the OpenAI Cookbook.

Haystack

Deepset's Haystack framework includes a Qdrant document store integration for building NLP pipelines with retrieval components.

AutoGen / CrewAI / Semantic Kernel

Integrations exist for Microsoft's agentic frameworks, enabling agent memory and tool-use patterns backed by vector search.

Qdrant Cloud

The managed service offering includes:

  • Managed clusters on AWS, GCP, or Azure
  • Auto-scaling based on workload demands
  • Terraform support via the Qdrant Cloud API
  • SSO and RBAC for enterprise authentication
  • Granular API keys with configurable permissions
  • Free tier for development and experimentation
