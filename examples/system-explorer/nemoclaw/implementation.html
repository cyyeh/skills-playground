Key Implementation Areas

Policy File Format (YAML)

▼

The openclaw-sandbox.yaml is the primary security configuration. It defines filesystem access, network egress rules, process limits, and seccomp profiles.

yaml 
apiVersion: nemoclaw/v1alpha1
kind: SandboxPolicy
metadata:
  name: openclaw-default

spec:
  filesystem:
    readWrite:
      - /sandbox
      - /tmp
    readOnly:
      - /usr
      - /lib
      - /etc
    denied:
      - /root
      - /var/run/docker.sock

  network:
    egress:
      default: deny
      allow:
        - host: "api.nvidia.com"
          ports: [443]
        - host: "huggingface.co"
          ports: [443]
          methods: ["GET"]
      ssrf_protection:
        block_private_ips: true
        resolve_dns: true

  process:
    max_processes: 64
    max_open_files: 1024

Tool Registration with TypeBox

▼

Tools use TypeBox for runtime-validated parameter schemas, ensuring type safety between the LLM's tool call and the execution function.

typescript 
import { Type, Static } from '@sinclair/typebox';

// Define typed parameters
const QueryParams = Type.Object({
  query: Type.String({
    description: "SQL query to execute",
    maxLength: 10000,
  }),
  database: Type.String({
    pattern: "^[a-zA-Z_][a-zA-Z0-9_]*$",
  }),
  timeout_ms: Type.Optional(Type.Number({
    minimum: 100,
    maximum: 30000,
    default: 5000,
  })),
});

// Register with the agent
api.registerTool({
  name: "database_query",
  description: "Execute a read-only SQL query",
  parameters: QueryParams,

  async execute(_id, params) {
    try {
      const result = await db.query(params.query, {
        database: params.database,
        timeout: params.timeout_ms ?? 5000,
        readOnly: true,
      });
      return {
        content: [{ type: "text",
          text: JSON.stringify(result) }],
      };
    } catch (error) {
      return {
        content: [{ type: "text",
          text: error.message }],
        isError: true,
      };
    }
  },
});

Error Handling and Retry Logic

▼

NemoClaw implements error handling at multiple levels with exponential backoff and provider fallback.

typescript 
async function routeInference(
  request: InferenceRequest,
  config: ProviderConfig,
): Promise<InferenceResponse> {
  const provider = selectProvider(request, config);

  for (let attempt = 0; attempt < 3; attempt++) {
    try {
      const response = await provider.complete(request);
      auditLog.record({
        type: 'inference',
        provider: provider.name,
        latency_ms: response.latency,
      });
      return response;
    } catch (error) {
      if (error.retryable && attempt < 2) {
        // Exponential backoff
        await delay(Math.pow(2, attempt) * 1000);
        continue;
      }
      // Attempt fallback provider
      if (config.routing.fallback) {
        return routeInference(request, {
          ...config,
          routing: { ...config.routing,
            default: config.routing.fallback,
            fallback: undefined },
        });
      }
      throw error;
    }
  }
}

Audit Log Format

▼

Every significant action generates a structured JSON audit entry for compliance review and operational monitoring.

json 
{
  "timestamp": "2026-03-31T10:15:23.456Z",
  "session_id": "sess_abc123",
  "event_type": "tool_call",
  "tool_name": "http_request",
  "parameters": {
    "url": "https://api.example.com/health",
    "method": "GET"
  },
  "policy_evaluation": {
    "rule": "egress.allow[1]",
    "result": "allowed",
    "latency_us": 45
  },
  "execution": {
    "status": "success",
    "response_code": 200,
    "latency_ms": 142
  }
}

Implementation Principle: NemoClaw's implementation follows a strict separation: security enforcement happens in compiled Rust (Policy Engine), orchestration happens in Python (Blueprint), the user interface happens in TypeScript/JavaScript (CLI plugin), and deployment automation happens in Shell scripts. Each language plays to its strengths.
