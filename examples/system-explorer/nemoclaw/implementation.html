Getting Started

The fastest path from zero to a running sandboxed OpenClaw instance requires a Linux machine (Ubuntu 22.04+), Docker, and a single command:

Bash
# Install NemoClaw
curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash

# Connect to your sandboxed agent
nemoclaw my-assistant connect

# Start the OpenClaw TUI inside the sandbox
openclaw tui

# Or send a single message
openclaw agent --agent main --local -m "Explain this codebase" --session-id test
ℹ️
Hardware minimums: 4 vCPU, 8 GB RAM, 20 GB disk. Recommended: 4+ vCPU, 16 GB RAM, 40 GB disk. The sandbox image alone is ~2.4 GB compressed. macOS (Apple Silicon) is supported via Colima or Docker Desktop. Windows requires WSL.

Code Patterns

Managing Sandbox Lifecycle

Bash
# List all sandboxes with model, provider, and policy details
nemoclaw list

# Check a specific sandbox's health
nemoclaw my-assistant status

# Stream logs in real time
nemoclaw my-assistant logs --follow

Managing Network Policies at Runtime

Bash
# View available policy presets
nemoclaw my-assistant policy-list

# Add a policy preset (e.g., allow PyPI)
nemoclaw my-assistant policy-add

# Apply custom policy without restart
openshell policy set custom-policy.yaml

# Monitor blocked requests in real time
openshell term

Deployment Considerations

⚠️
Upgrade path: Because NemoClaw is in alpha with breaking changes between releases, upgrades should be treated as full re-onboarding. Destroy the existing sandbox, update the CLI, and run nemoclaw onboard again.
💡
Monitoring: Use nemoclaw <name> logs --follow for real-time log streaming and openshell term for the interactive TUI showing network activity and pending approval requests.
