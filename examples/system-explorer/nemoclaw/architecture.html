Design Decisions Intermediate

Thin Plugin, Heavy Blueprint

The plugin handles user interaction only. All orchestration logic lives in the versioned blueprint. This means the CLI remains stable across major architectural changes, and blueprints can be independently released, verified, and rolled back.

Deny-by-Default Networking

Every outbound connection is blocked unless explicitly whitelisted. This is more restrictive than typical container networking but essential for autonomous agents that can write their own HTTP requests.

🔭
Architecture insight: The deny-by-default decision reflects a core principle: the alternative (allow-by-default) is unacceptable for always-on agents. The trade-off is operational friction -- operators must approve new endpoints -- but this is a cost worth paying for containment.

Credential Separation

API keys never enter the sandbox. The gateway injects credentials at the routing layer. A sandbox compromise cannot leak provider credentials. The trade-off is that inference always goes through the gateway (adding minor latency).

Kernel-Level Enforcement

Rather than relying on application-level sandboxing (which agents can potentially bypass), NemoClaw uses Landlock LSM, seccomp, and network namespaces -- kernel mechanisms that are immune to application-layer exploits.

ℹ️
Note: The kernel-level approach means Linux is the first-class platform. macOS and Windows require Docker Desktop with its own overhead, and some isolation guarantees are weaker on those platforms.

Directory Structure Intermediate

The codebase is organized into clearly separated host-side and sandbox-side components.

Text 
Host Environment
├── nemoclaw CLI (TypeScript plugin)
│   ├── bin/nemoclaw.js              # Entry point
│   ├── src/cli.ts                   # Commander.js subcommand wiring
│   ├── src/commands/                # launch, connect, status, logs, slash
│   └── src/blueprint/              # resolve, fetch, verify, exec, state
├── Blueprint Resolver
├── OpenShell Gateway
│   ├── Inference routing
│   ├── Credential management
│   └── Policy enforcement
└── Docker Runtime

Sandbox Environment (container)
├── OpenClaw Agent
├── NemoClaw Plugin (in-sandbox instance)
├── Inference endpoint (inference.local)
├── Landlock LSM
├── seccomp
└── Network namespace
