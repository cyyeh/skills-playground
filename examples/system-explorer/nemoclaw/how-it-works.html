How It Works

Sandbox Creation (Onboarding) Intermediate

The nemoclaw onboard command triggers a wizard that creates the full stack in one pass. Here is the sequence:

1
Blueprint Download

Resolve latest blueprint from OCI registry, verify digest

→
2
Resource Planning

Determine gateway, providers, sandbox, policy, route

→
3
OpenShell Provisioning

Pull sandbox image (~2.4 GB), create resources

→
4
Policy Applied

Deny-by-default networking and filesystem restrictions

→
5
Inference Configured

Provider registered, credentials stored host-side

After onboarding, the agent runs inside the sandbox with all isolation layers active.

Policy Enforcement Intermediate

NemoClaw enforces a two-tier policy model with different mutability characteristics:

LayerEnforcementMutability
Filesystem / ProcessLandlock LSM, seccomp, capability dropsLocked at creation -- immutable
Network / InferenceNetwork namespace, gateway routingHot-reloadable at runtime

This distinction is critical. Filesystem and process restrictions cannot be changed without destroying the sandbox. Network and inference policies can be updated dynamically:

Bash Policy hot-reload example View docs ↗ 
# Apply a preset policy to allow Slack access on a running sandbox
openshell policy set nemoclaw-blueprint/policies/presets/slack.yaml

Dynamic changes are session-scoped: when the sandbox stops, the running policy resets to the baseline.

💡
Tip: Start with generous preset policies for known integrations (Slack, Jira, PyPI), monitor the first few sessions, then add commonly needed endpoints to the baseline. This reduces operator approval fatigue significantly.

Inference Routing Intermediate

Inference requests from the agent never leave the sandbox directly. The agent only sees inference.local -- the real API credentials are injected host-side by the gateway.

1
Agent Calls Model

model.complete() targets inference.local

→
2
Plugin Intercepts

HTTP POST to gateway endpoint

→
3
Gateway Validates

Policy check + credential injection

→
4
Upstream Provider

NVIDIA, OpenAI, Anthropic, or local Ollama

Providers can be switched at runtime without restarting the sandbox:

Bash
openshell inference set --provider openai --model gpt-4o

Five-Layer Security Stack Intermediate

NemoClaw applies five protection layers, each addressing a different threat vector:

1
Container Isolation

Docker with hardened multi-stage base image

→
2
Process Hardening

Capability drops, seccomp syscall filtering

→
3
Filesystem Integrity

Root-owned configs, SHA256 verification, Landlock

→
4
Network Control

Deny-by-default egress, SSRF validation, operator TUI

→
5
Inference Security

Credential isolation, gateway routing, audit trail

🔭
Architecture insight: The key principle is defense in depth. No single layer is sufficient alone. A container escape is blocked by Landlock. A Landlock bypass is blocked by network namespaces. Each layer addresses what the others might miss.
