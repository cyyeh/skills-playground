Ecosystem & Integrations

Official Tools & Extensions

🛡️

NVIDIA OpenShell

The runtime foundation providing kernel-level sandboxing, network namespace isolation, and the inference routing gateway.

OpenShell is a general-purpose agent runtime that can host other agent frameworks beyond OpenClaw -- like how Docker can run any containerized application. NemoClaw is a purpose-built configuration layer on top of it.

🧠

NVIDIA Nemotron

NVIDIA's family of open-source language models, optimized for the NemoClaw inference pipeline. Includes Nemotron 3 Super 120B.

Can run locally via NVIDIA NIM or accessed through build.nvidia.com. Using Nemotron with NemoClaw provides the tightest integration because inference routing is pre-configured for NVIDIA's endpoints.

📦

NVIDIA NIM

A containerized inference runtime for self-hosting LLMs on NVIDIA hardware. Run the entire agent stack on-premises with no external API calls.

🧰

NVIDIA Agent Toolkit

The broader suite including OpenShell, NemoClaw, and other tools for building and deploying AI agents securely.

Community Ecosystem

⭐

awesome-nemoclaw

Community-curated collection of presets, recipes, and playbooks for sandboxed OpenClaw operations.

🔌

OpenClaw Extensions

The network policy includes an allowlist for ClawHub. Sandboxed agents can install and use OpenClaw extensions (MCP servers, custom tools) within the security boundary.

💬

Messaging Bridges

Official bridges for Telegram, Discord, and Slack. The architecture is simple enough for community members to build connectors for additional platforms.

Common Integration Patterns

💡
NemoClaw + Ollama (fully local inference): The privacy-maximizing configuration. Agent runs in a sandbox, all model calls go to local Ollama. No data leaves the machine. NemoClaw handles Ollama setup during onboarding including model pulling and warming.
ℹ️
NemoClaw + NVIDIA Endpoints (cloud-scale inference): For teams needing larger models (Nemotron 3 Super 120B) without local hardware. Sandbox still provides isolation while the cloud provides capabilities. Trade-off: inference data leaves the local machine (encrypted in transit).
💡
NemoClaw + Telegram/Slack (team-accessible agents): Connecting a messaging bridge transforms a sandboxed coding agent into a team resource. Multiple members interact through existing chat tools. The sandbox ensures network policy and approval workflow provide guardrails even for unexpected requests.
🔭
NemoClaw + GitHub (repository-aware agents): The default network policy allows access to GitHub's API and hosting endpoints. Enables common workflows: reading repos, reviewing PRs, creating issues -- all within the sandbox and still subject to the agent's GitHub credentials.
ℹ️
NemoClaw + custom proxies (enterprise model routing): Organizations with model gateways (LiteLLM, vLLM behind a reverse proxy) can point NemoClaw at their internal endpoint. NemoClaw validates by sending a real inference request, since custom proxies often lack standardized /models endpoints.
