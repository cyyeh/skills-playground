Common Integration Patterns

These are the proven ways teams combine NemoClaw with other tools and services.

NemoClaw + OpenClaw

The primary integration. NemoClaw wraps OpenClaw with security controls. OpenClaw provides the agent framework; NemoClaw provides the secure runtime. This is the default setup created by nemoclaw onboard.

NemoClaw + Local GPU (Nemotron / Ollama / vLLM)

For privacy-sensitive workloads. Run models locally on RTX/DGX hardware, with NemoClaw routing sensitive queries to local inference and general queries to cloud APIs. Best for compliance-heavy environments.

NemoClaw + Cloud APIs (Anthropic / OpenAI / NVIDIA)

For compute-flexible deployments. NemoClaw manages credential isolation and policy enforcement while using cloud models for inference. The fastest path to evaluation -- no local GPU needed.

NemoClaw + LangChain

NVIDIA has announced LangChain integration for building agent workflows that run inside NemoClaw sandboxes. Enables complex multi-step agent pipelines with full security containment.

NemoClaw + Enterprise Security Stack

Partnerships with Cisco, CrowdStrike, Google Cloud, Microsoft Security, and TrendAI for runtime policy management alignment. Bridges NemoClaw's agent-level controls with organization-wide security infrastructure.

💡
Tip: Start with the NemoClaw + Cloud API pattern for the fastest evaluation path. Add local GPU inference later if privacy requirements demand it. The credential isolation and policy enforcement work the same regardless of which provider you use.
