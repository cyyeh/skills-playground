Ecosystem & Integrations

Official tools, community resources, and the integration patterns that connect NemoClaw to the broader AI agent landscape.

Official Tools & Extensions

NemoClaw does not exist in isolation. It orchestrates and integrates with several NVIDIA-maintained projects that handle different parts of the agent infrastructure stack.

📦

OpenShell

The standalone sandboxed runtime that NemoClaw orchestrates. Can be used independently for other agents (Claude Code, Codex, GitHub Copilot). Provides container isolation, gateway, and policy enforcement.

Core Runtime
🧠

Nemotron Models

NVIDIA's open model family for local inference. Nemotron 3 Super 120B MoE (flagship) and Nemotron 3 Nano 4B (edge). Recommended for privacy-sensitive workloads on NVIDIA hardware.

Local Inference
🚀

AI-Q Blueprint

NVIDIA's development accelerator for agents, ranked top of DeepResearch Bench. Provides pre-built orchestration patterns and workflow templates that complement NemoClaw's security layer.

Agent Framework
⚙️

NVIDIA NIM

Local inference serving for NVIDIA models. Experimental integration with NemoClaw allows routing Privacy Router queries to NIM-hosted models on local GPU infrastructure.

Experimental

The Broader NVIDIA Agent Toolkit NemoClaw is one piece of NVIDIA's larger agent infrastructure strategy announced at GTC 2026. The full toolkit includes OpenShell for runtime sandboxing, Nemotron for local inference, AI-Q Blueprint for agent development, and NVIDIA NIM for model serving. NemoClaw ties these together with security orchestration, but each component is independently useful. OpenShell works with Claude Code, Codex, and Copilot -- not just OpenClaw.

How the Integrations Work

NemoClaw + OpenClaw

The primary integration and the reason NemoClaw exists. NemoClaw's Blueprint configures OpenShell specifically for OpenClaw agents, adding default network policies, the onboarding wizard (nemoclaw onboard), CLI commands, and the blueprint lifecycle. The agent framework itself is unchanged -- NemoClaw controls the environment it runs in, not how it thinks.

NemoClaw + Local GPU

For privacy-sensitive workloads where data must not leave your infrastructure. Run Nemotron models (or Ollama/vLLM) locally on RTX or DGX hardware. NemoClaw's Privacy Router automatically classifies each inference query and routes sensitive content (PII, proprietary code, financial data) to local models while sending general queries to cloud APIs for cost efficiency.

NemoClaw + Cloud APIs

For compute-flexible deployments where not all queries require local processing. NemoClaw supports Anthropic (Claude), OpenAI, Google Gemini, and any OpenAI-compatible or Anthropic-compatible endpoint. Credential isolation ensures API keys are stored on the host and injected by the Gateway at request time -- the agent never sees raw credentials, even when using cloud providers.

NemoClaw + LangChain

An announced but still-developing integration. LangChain provides the orchestration framework for building multi-step agent workflows (chains, tools, memory). NemoClaw provides the secure runtime where those workflows execute. The combination allows LangChain-based agents to run with kernel-level isolation and deny-by-default networking.

NemoClaw + Enterprise Security Stack

NVIDIA's partnerships with Cisco, CrowdStrike, Google Cloud, Microsoft Security, and TrendAI focus on aligning NemoClaw's runtime policy management with existing enterprise security tools. The goal is that NemoClaw policies can be managed and monitored through the same security dashboards and workflows that enterprises already use for their broader infrastructure.

🎯
The Key Takeaway NemoClaw is designed as a composable security layer, not a monolithic platform. It orchestrates OpenShell, integrates with multiple inference backends (local and cloud), and aligns with enterprise security stacks. The ecosystem is young but growing rapidly -- the community is filling gaps with deployment guides, policy presets, and tooling faster than NVIDIA ships official extensions.
