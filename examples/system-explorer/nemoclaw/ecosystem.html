Ecosystem & Integrations

Core Dependencies

🤖 OpenClaw

The open-source autonomous AI agent framework that NemoClaw wraps. OpenClaw provides task execution, tool use, memory, scheduled jobs, and messaging integrations. NemoClaw adds a security layer on top — your existing skills, configurations, and workflows carry over into the sandbox unchanged.

🛡️ NVIDIA OpenShell

NVIDIA's sandbox runtime for AI agents, part of the Agent Toolkit. Provides kernel-level isolation (Landlock, seccomp, network namespaces), inference routing gateway, and policy enforcement. OpenShell is agent-framework-agnostic; NemoClaw provides the OpenClaw-specific integration.

🧠 NVIDIA Nemotron Models

NemoClaw is optimized for NVIDIA's Nemotron model family. Ranges from the 4B-parameter Nano variant (edge) to the 120B-parameter Super variant (full local inference). Using Nemotron keeps all data on-premises, but NemoClaw also supports OpenAI, Anthropic, Gemini, and any OpenAI-compatible endpoint.

Messaging Platforms

NemoClaw connects sandboxed agents to messaging platforms through host-side bridge processes. Bridges run outside the sandbox, so messaging credentials never enter the agent's environment.

  • Telegram — Bot integration via host-side relay
  • Discord — Bot integration via host-side relay
  • Slack — App integration via host-side relay

Complementary Tools

📦 NanoClaw

Alternative lightweight agent framework using fully containerized, ephemeral Kubernetes-native environments. While NemoClaw wraps existing OpenClaw setups, NanoClaw builds security in from the ground up. Different architectural philosophy — evaluate both for your use case.

🔧 NVIDIA Agent Toolkit

The broader toolkit includes model optimization tools, inference servers (TensorRT-LLM, Triton), and evaluation frameworks. NemoClaw integrates naturally with other toolkit components for end-to-end agent deployment.

🔌 MCP (Model Context Protocol)

Standard protocol for AI agents to interact with external tools and data sources. OpenClaw supports MCP servers, and NemoClaw's network policy engine can be configured to allow connections to specific MCP endpoints while blocking unauthorized access.
