Should You Use NemoClaw? Beginner

Answer the questions below to find out if NemoClaw is the right fit for your use case.

Is NemoClaw Right For You?

Are you using (or planning to use) the OpenClaw agent platform?

Do you handle sensitive data (PII, proprietary code, financial data)?

NemoClaw is not a good fit. It is tightly coupled to the OpenClaw ecosystem. If you use LangChain, CrewAI, AutoGen, or other agent frameworks, consider OpenShell directly (the underlying runtime) or alternative sandboxing solutions.

Do you have Linux infrastructure (or Docker Desktop on macOS/Windows)?

Are you mainly experimenting, or do you need production-grade security?

NemoClaw is an excellent fit. You will get kernel-level isolation, credential separation, deny-by-default networking, and the full security stack. Start with curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash.
Consider carefully. NemoClaw works via Docker Desktop on macOS/Windows, but some kernel-level isolation guarantees (Landlock, seccomp) are weaker. If full isolation is critical, deploy on Linux.
Start with vanilla OpenClaw first. The 2.4 GB sandbox download and onboarding wizard are overkill for quick experiments. Add NemoClaw later when you need security.
NemoClaw is your best option. Even without sensitive data, deny-by-default networking and sandbox isolation protect against supply-chain attacks and accidental damage.

Real-World Scenarios Beginner

🛡️ Post-ClawHavoc Remediation

After the ClawHavoc supply-chain attack poisoned 1,184+ skills in OpenClaw's ClawHub, organizations adopted NemoClaw to sandbox deployments. Deny-by-default networking prevents malicious skills from exfiltrating stolen credentials, and blueprint digest verification prevents tampered artifacts from loading.

🏢 Local-First Enterprise Assistant

An organization running NemoClaw on NVIDIA DGX Spark hardware with Nemotron 3 Super 120B MoE achieves fully air-gapped AI assistance. No data leaves the local network. Sensitive queries are processed entirely on-premises.

💻 Developer Workstation Sandboxing

A developer runs NemoClaw on their Ubuntu workstation with OpenAI as the inference provider. The agent can write code in /sandbox, but cannot access ~/.ssh, ~/.aws, or any other sensitive host directories. Unknown network requests require explicit approval in the operator terminal.

ℹ️
Note: NemoClaw is alpha software. These scenarios represent intended use cases and early adoption patterns. Wait for a stable release before deploying in mission-critical production environments.
