Should You Use NemoClaw?

Use this interactive decision tree to determine whether NemoClaw is the right tool for your scenario.

Is NemoClaw Right for You?

Are you running an AI coding agent that executes code autonomously (without human review of every action)?

Do you need network egress controls to prevent the agent from contacting unauthorized endpoints?

Is your primary development platform Linux (or can you use Docker on Linux)?

Plain OpenClaw is sufficient. For interactive coding sessions where you review every action, NemoClaw's setup overhead is unnecessary. Use OpenClaw directly for a simpler, faster experience.
Consider a simpler Docker setup. If you do not need fine-grained network policy controls, a basic Docker container with --network=none may be sufficient. NemoClaw's value is in its policy engine and approval workflow.
NemoClaw is a strong fit. You need autonomous agent execution with network controls on Linux -- exactly NemoClaw's sweet spot. Start with curl -fsSL https://www.nvidia.com/nemoclaw.sh | bash and the onboarding wizard will guide you through setup.
NemoClaw works but with caveats. macOS/Windows support requires Docker Desktop or Colima, which adds a virtualization layer. The kernel-level isolation (Landlock, seccomp) runs inside the Linux VM, not natively. Expect some additional complexity.

When to Use NemoClaw

🤖

Always-On Coding Assistants

Run an AI agent that continuously monitors a codebase, responds on Slack or Telegram, and executes code -- with guarantees it cannot exfiltrate secrets or make unauthorized network calls.

🔍

Sandboxed Agent Testing

Run an agent in a deny-by-default environment and observe what endpoints it tries to reach. Incrementally grant access as you build trust -- like a probation period for a new employee.

🔒

Private Inference

Work with proprietary code that cannot leave your infrastructure. Use Ollama, NVIDIA NIM, or vLLM for local inference so model calls never leave your network.

💬

Multi-Channel Assistants

Make an AI assistant accessible from Telegram, Discord, and Slack while maintaining a single security boundary. All messages route through the same sandbox with the same policies.

When NOT to Use NemoClaw

⚠️
Quick interactive sessions: If you just want to chat with an AI about code for a few minutes, NemoClaw's setup overhead is unnecessary. Use plain OpenClaw.
⚠️
Production workloads (as of March 2026): NemoClaw is explicitly alpha software. APIs and runtime behavior may change without notice. Treat it as an evaluation tool.
ℹ️
Teams without Docker: NemoClaw requires Docker or a compatible container runtime. The setup overhead is significant for teams not already using containers.

Real-World Examples

📝

Enterprise Code Review

A sandboxed OpenClaw agent connected to Slack reviews code snippets. It accesses private GitHub repos (allowed in policy) but cannot reach any other external services without operator approval.

🔬

Secure Research Lab

Researchers use NemoClaw with local Ollama inference -- no model calls leave the network. The deny-by-default policy blocks attempts to upload data to cloud services.

🛠️

DevOps with Human-in-the-Loop

An agent monitors infrastructure logs and suggests remediation. It can read dashboards (allowed endpoints) but cannot execute changes without operator approval first.
