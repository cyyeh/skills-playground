Should You Use NemoClaw?

Not every project needs kernel-level agent security. Use this interactive decision tree to find out if NemoClaw is the right fit for your use case.

Is NemoClaw Right for You?

Are you deploying autonomous AI agents that run without continuous human supervision?

Do your agents access sensitive systems -- production APIs, customer data, or internal infrastructure?

NemoClaw is likely overkill. If your agent runs a single task under human supervision and does not access sensitive systems, just use OpenClaw directly. NemoClaw's security infrastructure adds complexity that is not justified for supervised, low-risk tasks.

Are you running on Linux, or can you provision a Linux server?

NemoClaw is a good fit for experimentation. Even without sensitive data, the sandbox provides a safe playground for exploring what autonomous agents can do without risking your host system. Start with cloud inference for the easiest setup.

Do you need production-grade stability right now?

Wait for better platform support. NemoClaw is Linux-first. macOS has partial support with known bugs, WSL2 is experimental, and Intel Macs are unsupported. Wait for beta if you cannot provision a Linux environment.
Not yet -- evaluate for Q3 2026. NemoClaw is alpha software with known CVEs and missing features. Use it now for evaluation and architecture planning, but wait for at least beta stability before committing to production.
NemoClaw is an excellent fit. You have the right infrastructure (Linux), genuine security needs (autonomous agents + sensitive systems), and realistic expectations (evaluation, not production). Deploy it today for security research, architecture evaluation, and agent experimentation.

When to Use It

Always-On Enterprise Agents

Agents running 24/7 with real security guarantees when accessing production systems, customer data, or internal APIs.

Compliance-Sensitive Deploys

When regulations require certain data never leaves your infrastructure. The Privacy Router keeps PII on local models automatically.

Secure Experimentation

Explore what autonomous agents can do without risking your host system. A rogue agent cannot damage anything outside /sandbox.

Multi-Model Inference

Route to the best model per task -- local Nemotron for private data, cloud Claude/GPT for general tasks -- with automatic sensitivity routing.

Security Architecture Reference

Study a well-documented, open-source reference implementation of agent sandboxing, even if you build your own system.

When NOT to Use It

Production Workloads Today

Alpha software with known CVEs. NVIDIA explicitly warns it is not production-ready. Evaluate for production in Q3 2026 at earliest.

macOS or Windows Primary

Linux-first. macOS has partial support with known issues. WSL2 is experimental. Intel Macs are unsupported entirely.

Low-Resource Environments

Minimum 8 GB RAM and 20 GB disk for just the infrastructure. Not suitable for lightweight VMs or constrained hardware.

Multi-Tenant Isolation

Single-sandbox, single-operator model. Cannot isolate agents from each other within a shared instance.

💡
Tip: If you just need basic sandboxing without agent-specific security controls, consider running agents in plain Docker containers. NemoClaw's value is in the agent-aware features: per-binary network policies, inference routing with credential isolation, and the real-time approval TUI.

Real-World Examples

GTC 2026 "Build-a-Claw" Event -- NVIDIA demonstrated NemoClaw live at GTC 2026 with sandbox creation and agent deployment, showcasing the onboarding wizard and real-time network approval flow for a coding agent.

Community Deployments -- Published guides for running NemoClaw on DigitalOcean Droplets, DGX Spark, and via NVIDIA Brev for remote GPU access, indicating active experimentation across different infrastructure setups.
