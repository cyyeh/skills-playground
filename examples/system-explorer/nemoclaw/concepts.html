Core Concepts

The Seven Building Blocks Beginner

NemoClaw is built from seven interlocking components. Each one handles a specific aspect of agent security and orchestration. Click "Learn more" on any card to dive deeper.

🏦

Sandbox

An isolated container where the OpenClaw agent runs, restricted to specific filesystem paths, network endpoints, and system calls.

Think of it like a bank vault with a teller window. The agent can interact with the outside world, but only through approved channels. Even a compromised agent cannot override its own restrictions because enforcement happens outside the agent entirely.

📐

Blueprint

A versioned, cryptographically verified Python artifact containing sandbox creation logic, policies, and inference config.

Like an architect's sealed building plan. Before sandbox creation begins, the plan is verified against a tamper-proof cryptographic digest. This prevents supply-chain attacks where someone substitutes a tampered artifact.

🔐

OpenShell Runtime

NVIDIA's secure runtime providing kernel-level isolation using Landlock LSM, seccomp, and network namespaces.

OpenShell is the prison warden. The sandbox is the cell, the policies are the rulebook. Security controls are enforced at the OS kernel level, which means no amount of prompt injection or agent cleverness can bypass the restrictions.

🛡️

Network Policy

Declarative YAML rules governing which endpoints the sandbox can reach. Deny-by-default: anything not explicitly allowed is blocked.

Like a corporate firewall with a whitelist. The agent cannot call home, download packages, or exfiltrate data unless every destination is pre-approved. Unknown requests get escalated to a human operator for real-time approval.

📨

Inference Router

A gateway that intercepts all model API calls and routes them to configured providers. Credentials stay on the host.

Like a mail room in a classified facility. Outgoing letters go through a central desk that stamps them with the right postage (API credentials) and routes them to the correct recipient. The sender never sees the postage meter or address book.

🔌

Plugin

A lightweight TypeScript module in the OpenClaw CLI that registers commands and manages the blueprint lifecycle.

The receptionist at the front desk. It takes your request, routes it to the right department (blueprint, sandbox, policy engine), and reports back. The plugin-blueprint separation ensures commands remain stable even as the backend evolves.

📺

Operator Terminal

A TUI for monitoring sandbox activity in real time and approving or denying blocked network requests.

Like an air traffic control tower. The operator watches all outbound "flights" from the sandbox, approves cleared ones, and blocks anything suspicious -- all without the agent being able to override the decision.

How They Fit Together Beginner

The system operates in concentric layers. At the center, the OpenClaw agent runs inside a sandbox, which is built from a verified blueprint.

The sandbox communicates with the outside world through two controlled channels: the inference router (for model calls) and the network policy (for everything else). The OpenShell runtime enforces all policies at the kernel level.

Why Agent Security Matters Beginner

Autonomous AI agents that can write code, spawn sub-agents, and persist across sessions create security risks that traditional containers do not fully address. The ClawHavoc supply-chain attack (February 2026) poisoned 1,184+ skills in OpenClaw's ClawHub registry, proving that the ecosystem needed stronger containment.

⚠️
Warning: Running autonomous agents without sandboxing is like giving a stranger admin access to your machine. They might be helpful, but if compromised, they can exfiltrate credentials, modify system files, and make unauthorized API calls.

NemoClaw's design philosophy is that security controls must be mathematical runtime properties, not model personality traits. You cannot rely on an AI model to self-regulate; you must enforce restrictions at a level the model cannot reach -- the operating system kernel.
