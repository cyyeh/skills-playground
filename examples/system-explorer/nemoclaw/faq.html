NemoClaw currently operates in “single-player mode” — one sandbox per gateway, one trusted operator per gateway. There is no multi-tenant isolationThe ability to securely run multiple users' workloads on shared infrastructure, guaranteeing that one tenant cannot access another tenant's data or resources., no horizontal scaling of sandboxes, and no centralized management of multiple NemoClaw instances.

For teams needing dozens of sandboxed agents, you would need to run separate NemoClaw instances, each with its own gateway and policy configuration. NVIDIA has indicated multi-tenant support is on the roadmap but has not committed to a timeline.

💡
Containment Reduces Blast Radius

NemoClaw does not prevent prompt injection — no system can guarantee that today. What it does is ensure that even a fully compromised agent is contained: restricted filesystem writes, no raw credential access, and only pre-approved network endpoints. The damage a prompt-injected agent can do inside NemoClaw is orders of magnitude less than one running uncontained on your host system.
