Frequently Asked Questions

01 Does NemoClaw replace OpenClaw?

No. NemoClaw is a complementary security layer, not a replacement. It wraps an existing OpenClaw installation inside a hardened sandbox with managed inference and network controls. Your existing OpenClaw agent — skills, memory, configuration, messaging integrations — runs unchanged inside the NemoClaw sandbox.

02 Can an agent bypass the sandbox security controls?

The controls operate at the kernel level, outside the agent's process. Landlock policies are locked before the agent starts. seccomp filters are locked at container creation. The network namespace gives the agent a completely separate network view.

However, no sandbox is perfectly escape-proof. Kernel vulnerabilities or container runtime escapes could theoretically bypass controls. NemoClaw provides defense-in-depth, not absolute guarantees. Regular updates and security audits remain essential.

03 How much latency does inference routing add?

The gateway proxy hop adds single-digit milliseconds of overhead. Actual latency is dominated by the inference provider's response time (hundreds of milliseconds to seconds). For most use cases, the routing overhead is negligible compared to model inference time.

04 How do credentials stay safe if the agent is compromised?

API credentials are stored on the host at ~/.nemoclaw/credentials.json and never copied into the sandbox. The agent talks to inference.local; the gateway attaches credentials on the host side. Even with code execution inside the sandbox, the attacker cannot:

  • Read the credentials file (Landlock blocks access)
  • See the host filesystem (container isolation)
  • Intercept the gateway's credential attachment (different network namespace)
05 What happens when the agent needs a new network endpoint?

The deny-all policy blocks the connection. The blocked request appears in the operator's TUI (openshell term) with full context: destination host, port, and requesting binary. The operator approves or denies in real time. Approved endpoints persist for the session; permanent changes require editing the YAML policy.

Preset policy files exist for common integrations (PyPI, Docker Hub, Slack, Jira).

06 Does NemoClaw work on macOS or Windows?

Partially. The full security stack (Landlock, seccomp, network namespaces) requires Linux. On macOS with Apple Silicon, NemoClaw runs via Colima or Docker Desktop but with weaker isolation. On Windows, WSL2 with Docker Desktop is required. For production with full guarantees, Linux is the recommended platform.

07 Is NemoClaw production-ready?

No. As of March 2026, NemoClaw is in alpha (early preview). NVIDIA explicitly states it is not production-ready and APIs may change without notice. Use it for evaluation, development, and testing — not for production workloads with sensitive data.

⚠
The project has been publicly available for approximately two weeks. There are no production deployment case studies and no independent security audits have been published.
08 What is the Privacy Router?

The Privacy Router classifies each query by data sensitivity. Queries containing PII, proprietary code, or financial data are routed to a local Nemotron model (never leaving your infrastructure). Non-sensitive queries go to cloud models for better capability. This lets teams balance privacy with model performance.

09 Can I use NemoClaw with non-OpenClaw agents?

NemoClaw is designed specifically for OpenClaw. The CLI, blueprint, and onboarding all assume OpenClaw as the runtime. However, the underlying OpenShell runtime is agent-framework-agnostic. For other frameworks, use OpenShell directly without NemoClaw.

10 How does blueprint versioning prevent supply-chain attacks?

Each blueprint artifact is immutable and identified by a cryptographic digest. The plugin checks compatibility with installed OpenShell/OpenClaw versions and verifies the digest before applying updates. If the digest does not match, the update is rejected. Re-running onboard with a new version recreates the sandbox from scratch — there is no in-place upgrade.
