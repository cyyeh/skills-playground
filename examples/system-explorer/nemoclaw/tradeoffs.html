Trade-offs & Limitations

Strengths

✅ Defense-in-Depth Security

Layers Landlock, seccomp, network namespaces, and inference routing into a multi-barrier architecture. Compromising any one layer still leaves the others intact.

✅ Zero-Change Migration from OpenClaw

Existing setups can be wrapped without modifying agent skills, configurations, or messaging integrations, significantly lowering the adoption barrier.

✅ Operator-Visible Policy Enforcement

Every blocked action is surfaced in real time via the TUI. The approval workflow provides transparency and control over what the agent can do.

✅ Reproducible, Auditable Sandboxes

Blueprint model guarantees every sandbox is built from the same immutable, digest-verified specification. Security audits are straightforward.

✅ Apache 2.0 License

Open source with a permissive license — freely adopt, modify, and integrate into commercial products.

Limitations

❌ Linux-Only for Full Security

Complete Landlock/seccomp/netns stack only works on Linux. macOS and Windows WSL deployments have weakened isolation.

❌ NVIDIA Ecosystem Gravity

Optimized for Nemotron models and DGX hardware. Privacy Router tightly integrated with Nemotron. Non-NVIDIA setups get fewer features.

❌ Early Alpha Maturity

Publicly available for approximately two weeks. No production case studies, no independent security audits, APIs may change without notice.

❌ Operational Overhead

Adds a gateway process, sandbox container, host-side state files, policy management, and bridge processes. May not be justified for simple deployments.

❌ Single-Node Only

No built-in horizontal scaling, multi-region failover, or high-availability clustering. Must build your own orchestration for scale.

Key Risks to Evaluate

1. Vendor Lock-in Trajectory

Deep integration with NVIDIA hardware and models creates migration friction. If NVIDIA changes licensing or strategy, switching may require significant rework.

2. Kernel Security Assumptions

Security depends on Linux isolation primitives. Landlock is relatively new — kernel vulnerabilities could undermine the entire model. No independent audits published yet.

3. Single-Node Limitation

No horizontal scaling support. Organizations needing multi-region failover or HA must build their own orchestration layer.

4. Rapid API Changes

Alpha-stage APIs, configs, and runtime behavior subject to breaking changes. Investment in automation or tooling carries rework risk.
